# 图片搜索与以图搜图

配合 `astrbot_plugin_serpapi_imgsearch` 的两个函数工具使用。收到搜图 / 以图搜图需求时**直接**调用下述工具，勿先调用其他泛检索工具（如 `web_search_tavily`、`grok_web_search`）。

## 可用工具

- **`pic_search`（文字搜图）**：按关键词搜图并直接发给用户，抓取、筛选、发送均在工具内完成。`query` 传从用户话语提取的核心关键词，勿润色扩写。
  ```json
  {"name": "pic_search", "arguments": {"query": "赛博朋克 城市"}}
  ```
- **`reverse_image_search`（以图搜图）**：反向检索图片来源，自动读取当前 / 被引用消息中的图片。`image_url` 一般留空，仅在已有公网直链时才传。
  ```json
  {"name": "reverse_image_search", "arguments": {}}
  ```

## 使用要点

**文字搜图**：调用 `pic_search` 即可，工具会自行发图，完成后无需告知用户，返回 `error:` 则转达原因。

**以图搜图**（视觉与检索双通道互相印证）：
1. **先视觉辨识**：解析构图、画风、角色、文字等特征。
2. **再反向检索**：调用 `reverse_image_search` 获取来源。
3. **引擎强制印证**：无论你对结果有多确信，只要从视觉或反查中提取到了具体的角色名、作者或作品名，**必须调用网页搜索/网页提取进行交叉核实**，坚决避免幻觉。
4. **结构呈现**：纯文本分段输出——视觉辨识 / 反查来源（标题+来源+直链） / 引擎核实（展示交叉比对的证据） / 最终结论。**必须直接输出完整 URL，绝对禁止使用任何 Markdown 格式（如 `[]()` 或 `![]()`）**。
5. **失败降级**：若反查无果，勿放弃，立即降级为纯视觉推理并说明情况。

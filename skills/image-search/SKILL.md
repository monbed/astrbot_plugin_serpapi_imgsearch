# 图片搜索与以图搜图 (Image Search & Reverse Image Search)

本技能配合 `astrbot_plugin_serpapi_imgsearch` 插件提供的两个函数工具使用。收到搜图 / 以图搜图需求时，必须**直接**调用下述专用工具，严禁先调用其他泛检索工具（如 `web_search_tavily`、`grok_web_search` 等）。

## 可用工具

### 1. 文字搜图：`pic_search`
根据关键词用 Google 图片（SerpApi `google_images`）抓取大量候选图，再用视觉模型多轮筛选出最匹配的一张并**直接发送给用户**（工具内部完成抓取、筛选、发送）。

* `query`（必填）：从用户需求中**直接提取的核心搜索关键词**，不要过度润色或自行扩写，保持简洁。例如用户说"来张赛博朋克城市的图"，传 "赛博朋克 城市" 即可。
* **调用示例**：
  ```json
  {"name": "pic_search", "arguments": {"query": "赛博朋克 城市"}}
  ```

### 2. 以图搜图：`reverse_image_search`
对图片做 Google Lens（SerpApi `google_lens`）反向检索，返回匹配的网页来源。**会自动读取当前消息或被引用消息中的图片**并转为公网链接。

* `image_url`（可选）：仅当你已拥有图片的公网 http/https 直链时才传；否则留空。
* **调用示例**：
  ```json
  {"name": "reverse_image_search", "arguments": {}}
  ```

## 核心执行流程与硬性约束

### 文字搜图
1.  收到"找图 / 搜图 / 发一张 xx 的图"需求时，直接调用 `pic_search`，传入从用户话语中提取的**核心关键词**（不要过度润色或扩写）。
2.  工具会自行抓取候选图、用视觉模型筛选并把最佳图发送给用户——你**无需**再调用其他发送工具。
3.  工具返回成功后，用简短自然语言告知用户已发送即可；若返回 `error:` 开头的内容，请把原因转达用户并建议调整关键词后重试。

### 以图搜图（视觉与搜索双通道互相印证）
1.  **自主视觉辨识**：先完全依靠你的多模态视觉能力，解析图片的构图、色彩、画风、签名、角色、文字等，形成初步推论。
2.  **反向检索**：直接调用 `reverse_image_search`。
3.  **深度交叉印证**：**【核心步骤】** 把"第 1 步的视觉特征"与"第 2 步的搜索结果"强力比对：用视觉细节验证搜索结果真实性，用搜索文本精准定位视觉推论（作者名、作品标题、发布链接等）。
4.  **结构化反馈**：工具不主动发图，结果以 JSON（含 `items` 与 `instruction`）返回，全部由你用纯文本呈现。整理为三个板块——视觉特征辨识 / 反向检索比对（标题 + 来源 + 完整链接）/ 交叉印证结论（最可能的作者 / 作品名 / IP 出处）。直接输出完整 URL，勿用 Markdown 链接语法（部分平台不支持）。
5.  **容错降级（纯视觉推导）**：若 `reverse_image_search` 返回错误（无图、无法转公网链接、无结果等），**严禁放弃**，必须立刻降级为**纯视觉辨识与推理**，把你对该图的全部推导直接呈现给用户，并坦诚说明这是基于视觉的推测。
